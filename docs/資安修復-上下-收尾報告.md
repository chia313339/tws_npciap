# 資安弱點修復收尾報告 — 上下(主站 Vue SPA）

> 對象：本案「上下」負責的主站 `ntpc-ai.ntpc.gov.tw`（Vue SPA）。
> 用途：記錄本站每一項資安弱點「改在哪、怎麼改、怎麼驗」，供**重掃複測、重新部署、維運交接、下次演練**重複使用。
> 業主的完整弱點 SOP（標明機密、含內部 IP/全弱點清單）**不納入版控**，請另存於本機／加密管道。
> 最後更新：2026-06-03。相關文件：[csp-allowlist.md](./csp-allowlist.md)。

## 0. 架構與責任邊界（先看懂這個，後面才好懂）

```
公網 443
  └─ NPM 反向代理容器（Nginx Proxy Manager，OpenResty 底）  ← SSL 終結 + 網域綁定 + 安全標頭都在這
       └─ Vue SPA 容器（nginx:alpine，內部 8080→80）        ← 前端打包產物 + SPA fallback / 404 規則
```

- 業主 SOP 裡寫的「**主站 openresty**」**就是我們自己的 NPM 容器**（NPM 底層是 OpenResty），不是台智雲的某台機器。
- 因此 **HTTP 標頭類弱點（CSP、x-served-by、ETag…）是改在 NPM**，不是改這個 repo、也不是台智雲。
- **前端／應用層弱點（SRI、robots.txt、SPA 404）才是改這個 repo**（[index.html](../index.html)、[nginx.conf](../nginx.conf)、[public/](../public/)）。

| 修改位置 | 對應弱點 | 持久性 |
|---|---|---|
| NPM → Proxy Host（主站）→ **Advanced → Custom Nginx Configuration** | VULN-004 CSP、安全標頭群 | 持久（NPM 的 `/data` 有掛載卷） |
| NPM 容器 `/etc/nginx/conf.d/*`（`add_header X-Served-By`） | VULN-015 | ⚠️ **非持久**，見 §7 |
| repo [index.html](../index.html) | VULN-011 SRI | 隨容器部署 |
| repo [nginx.conf](../nginx.conf) | VULN-012 404、VULN-016 ETag | 隨容器部署 |
| repo [public/robots.txt](../public/robots.txt)、[public/sitemap.xml](../public/sitemap.xml) | VULN-013 | 隨容器部署 |

## 1. 各弱點修復明細（上下範圍，2026-06-03 全部驗證上線）

### VULN-004 — CSP 由 Report-Only 改為強制（Medium）
- **改在**：NPM 主站 Proxy Host 的 Advanced 自訂設定（`more_set_headers`）。
- **做法**：標頭名稱 `Content-Security-Policy-Report-Only` → `Content-Security-Policy`，**並補上 GA 網域**。
- **重點坑**：業主 SOP §1 的 CSP **漏了 `googletagmanager.com` 與 `google-analytics`**，照抄會把 GA4 擋死。正確白名單見 [csp-allowlist.md](./csp-allowlist.md)；實際採用的強制版政策已含：
  - `script-src … https://www.googletagmanager.com https://cos.twcc.ai …`
  - `connect-src … https://www.googletagmanager.com https://www.google-analytics.com https://*.analytics.google.com https://*.google-analytics.com wss://edb.ntpc-ai.ntpc.gov.tw …`
- **驗證**：`curl -sI https://ntpc-ai.ntpc.gov.tw/ | grep -i content-security` → 只有 `content-security-policy:`（無 `-report-only`）、且含 `googletagmanager`。
- **強制前後務必用「無痕（無擋廣告擴充）瀏覽器」實測**（webchat、GA 即時報表、字型），避免擴充擋掉 GA 而誤判。

### VULN-011 — 外部 script 補 SRI（Low）
- **改在**：[index.html](../index.html) 的 webchat `<script>`，加 `integrity="sha384-…" crossorigin="anonymous"`。
- **GTM 不加 SRI**（Google 動態 script，固定 hash 會在 Google 改版時壞掉）→ 改靠 CSP 白名單管控，掃描器若標記就以「Google 官方動態資源，採 CSP 白名單」說明（業界標準）。
- **驗證**：`curl -s https://ntpc-ai.ntpc.gov.tw/ | grep -o 'integrity="sha384-[^"]*"'`。

### VULN-012 — 不存在／敏感路徑回真實 404（Low）
- **改在**：[nginx.conf](../nginx.conf)，敏感副檔名與路徑回 404，其餘維持 SPA fallback（deep-link 仍可用）。
- **驗證**：`/test.git` → 404、`/about` → 200。

### VULN-013 — 加真實 robots.txt（Low）
- **改在**：[public/robots.txt](../public/robots.txt)（build 後在 `dist/robots.txt`），另含 [public/sitemap.xml](../public/sitemap.xml)。
- **驗證**：`curl -s -o /dev/null -w "%{content_type}\n" https://ntpc-ai.ntpc.gov.tw/robots.txt` → `text/plain`。

### VULN-015 — 移除 x-served-by（Info）
- **來源**：NPM 容器內 `/etc/nginx/conf.d/include/proxy.conf` 與 `/etc/nginx/conf.d/production.conf` 各有一行 `add_header X-Served-By $host;`（非 NPM 預設，是人為加的）。
- **做法**：移除那兩行後 reload。`more_clear_headers` 在 server 層**壓不掉 location 層的 `add_header`**（過濾器順序問題），所以必須從源頭刪。
- **驗證**：`curl -sI https://ntpc-ai.ntpc.gov.tw/ | grep -i x-served-by` → 無輸出。
- ⚠️ **持久性注意**，見 §7。

### VULN-016 — 關閉 ETag（Info）
- **改在**：[nginx.conf](../nginx.conf) 的 `etag off;`（縱深防禦；反代層亦同步）。
- **驗證**：`curl -sI https://ntpc-ai.ntpc.gov.tw/ | grep -i '^etag:'` → 無輸出。
- 小提醒：grep `etag` 會誤命中 CSP 字串裡的「googl**etag**manager」，驗證時請用 `^etag:` 錨點。

## 3. 一鍵總驗（重掃前自我檢查／當證據）

```bash
echo "== 主要標頭（x-served-by / etag 應為空）=="
curl -sI https://ntpc-ai.ntpc.gov.tw/ | grep -iE '^(server|content-security-policy|x-served-by|etag|strict-transport-security):'
echo "== robots =="
curl -s -o /dev/null -w "robots.txt content-type=%{content_type}\n" https://ntpc-ai.ntpc.gov.tw/robots.txt
echo "== 404 / SRI =="
curl -s -o /dev/null -w "/test.git => %{http_code}\n" https://ntpc-ai.ntpc.gov.tw/test.git
curl -s https://ntpc-ai.ntpc.gov.tw/ | grep -o 'integrity="sha384-[^"]*"' | head -1
```

預期：`server: openresty`（無版號）、CSP 為強制版含 googletagmanager、無 x-served-by、無 etag、robots 為 text/plain、`/test.git`=404、首頁有 `integrity=`。

## 4. 重新部署前端容器（修復若只在 git／dist、尚未上線時）

> 教訓：**修復 commit 進 git ≠ 已上線**。一定要用上面的 curl 對「正式機」驗，不能只看 git 狀態。

```bash
cd ~/<repo>            # VM 上的專案目錄
git pull              # 確認含最新修復 commit
npm ci && npm run build
docker build -t twcc:<新tag> .
docker inspect twcc --format 'Net={{range $k,$v := .NetworkSettings.Networks}}{{$k}} {{end}}| Ports={{.HostConfig.PortBindings}} | Restart={{.HostConfig.RestartPolicy.Name}}'
# 用上面看到的「相同設定」重建容器（埠/網路/restart 要一致，否則 NPM 連不到上游）
docker stop twcc && docker rm twcc
docker run -d --name twcc --restart unless-stopped -p 8080:80 twcc:<新tag>
```
舊 image 保留可隨時回滾。換容器有數秒空窗，建議低流量時段。

## 5. CSP 異動 SOP（之後要加／改外部來源時）

1. NPM → Proxy Host（主站）→ Advanced，找到 `more_set_headers "Content-Security-Policy: …"`。
2. 把要新增的來源加進對應 directive（script/style/img/font/connect/frame…）。
3. 高風險時先改回 `-Report-Only` 觀察 console，無紅字再強制。
4. Save 後 `curl -sI … | grep -i content-security` 驗證；無痕瀏覽器實測全功能。
5. 來源清單與出處對照見 [csp-allowlist.md](./csp-allowlist.md)。

## 7. ⚠️ 待辦：x-served-by 的修改「不持久」

NPM 容器只掛載了 `/data` 與 `/etc/letsencrypt`，**`/etc/nginx` 沒有掛載**。VULN-015 的移除是改在 `/etc/nginx/conf.d/*`：

- `docker restart npm`：**沒事，修改還在**。
- `docker rm` 重建 npm（例如升級 NPM）：`/etc/nginx` 會還原成映像檔版本。標準 NPM 映像**不會**送 X-Served-By，所以多半維持乾淨；但若有 provisioning 腳本會重新加，就會復發。

**建議**：
1. 演練前**不要重建 npm 容器**（重啟可以）。
2. 確認無腳本會重新注入：`grep -rin "X-Served-By" ~ 2>/dev/null`。
3. 要徹底固化：把修正後的 `/etc/nginx/conf.d/*` 改為 bind mount 或納入自訂映像（演練後處理）。
